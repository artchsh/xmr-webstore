# Project Memory — 2026-03-10

## Project Vision & Goals

- Build a privacy-first, self-hostable Monero-only digital goods webshop for a single seller.
- Keep operations simple for low-to-moderate technical users (CLI-capable, not necessarily Docker/VPS experts).
- Prioritize security/privacy and deterministic server-rendered flows over frontend complexity.
- Maintain minimal infrastructure: SQLite, FastAPI SSR, Caddy, wallet-rpc, Tor.
- Ensure deployability within ~30 minutes on a VPS.

## Core Architecture & Tech Stack (with exact versions)

### Runtime architecture (5 containers)

1. `webshop` — public SSR storefront (FastAPI)
2. `admin` — separate SSR admin service (FastAPI)
3. `wallet-rpc` — internal Monero wallet RPC service
4. `caddy` — reverse proxy + HTTPS edge
5. `tor` — SOCKS5 proxy for outbound privacy routing

### Core software versions

- Python base image: `python:3.12-slim`

## Key Decisions & Trade-offs

- **SSR over SPA/API-first UX:** reduces attack surface and complexity for MVP.
- **SQLite over server DBs:** simplest self-host model for one-seller deployment.
- **No customer accounts/email:** stronger privacy posture, less operational burden.
- **Separated admin service:** stronger boundary and safer default exposure model.
- **Monero-only with wallet-rpc:** focused payment path, no multi-chain complexity.
- **Node failover support:** practical resilience against unstable community nodes.
- **No automatic favicon/PWA icon generation yet:** deferred to avoid image-processing complexity in MVP.
